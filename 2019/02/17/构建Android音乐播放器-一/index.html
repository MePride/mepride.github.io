3.播放音乐

MediaPlayer是一个封装好的音频、视频流媒体操作类，其内部都是调用native方法，由C++实现的。

作为一个流媒体操作类，那么必然涉及到了播放、暂停、停止等操作。

void start(); //开始或者恢复播放

void stop(); //停止播放

void pause(); //暂停播放

通过上面三个方法，只要设置好音频源，即可在应用中播放流媒体资源，除此之外，MeidaPlayer还为我们提供了一些其他的方法，这里列出一些常用的，更多的参考官方文档。

int getDuration(); //获取流媒体的总播放时长，单位是毫秒

int getCurrentPosition(); //获取当前流媒体播放的位置

void seekTo(int msec); //设置当前播放位置，单位是毫秒

void setLooping(boolean looping); //设置循环播放

boolean isLooping(); //判断是否在循环播放

boolean isPlaying(); //判断是否在播放

void prepare(); //同步流媒体文件

void prepareAsync(); //异步的方式装载流媒体文件

void release(); //回收流媒体资源

void setAudioStreamType(int streamtype)//设置播放流媒体类型

void setWakeMode(Context context,int mode)//设置CPU唤醒的状态

setNextMediaPlayer(MediaPlayer next); //当前流媒体播放完,下一个播放的MediaPlayer

4.回调方法

  • setOnCompletionList4ener(MediaPlayer.OnCompletionListener listener);//当流媒体播放完毕的时候回调。

  • setOnErrorListener(MediaPlayer.OnErrorListener listener);//当播放中发生错误的时候回调。

  • setOnPreparedListener(MediaPlayer.OnPreparedListener listener);当装载流媒体完毕的时候回调。
  • setOnSeekCompleteListener(MediaPlayer.OnSeekCompleteListener listener);当使用seekTo()设置播放位置的时候回调。
5.回收资源

MediaPlayer是很消耗系统资源的，所以使用完后最高好主动回收，不要等待自动回收。

二、Service

Service是Android四大组件直以，是Android中实现程序后台运行的解决方案，它常用来执行那些不需要和用户交互且需要长期运行的任务。Service独立运行在一个进程中，类似UI线程，所以最好不要在Service中执行耗时操作，除非在Service中创建子线程完成耗时操作，Service运行不依赖任何用户界面，即使程序被切换到后台或者用户打开别的程序,Service仍然能够保持正常运行。若是某个应用程序的进程被清理，所有依赖于该进程的Service也会停止。

3.Service的通信方式

  • 通过Intent

    缺点：与UI层难以交互

  • 通过onBind

    缺点：不能后台执行

    • 远程服务AIDL

      AIDL（Android Interface Definition Language）是Android接口定义语言的意思，它可以用于让某个Service与多个应用程序组件之间进行跨进程通信，从而可以实现多个应用程序共享同一个Service的功能。实际上实现跨进程之间通信的有很多，比如BroadCast，但是AIDL的优势在于速度快(系统底层直接是共享内存)，性能稳，效率高，一般进程间通信就用它。
